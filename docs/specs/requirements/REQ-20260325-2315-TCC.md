# REQ: Tactical Command Center (TCC) 統合要件

## 1. ビジョンとテーマ
単なるタスク管理・進捗メモアプリを超え、ナレッジワーカーのPC起動時のファーストアクションとなる**「統合戦術管制システム（Tactical Command Center）」**を構築する。
- **UIコンセプト**: フライトシューティング（Ace Combat等）の「戦術データリンク」を彷彿とさせる、ダークブルー基調で情報密度の高いサイバーパンクUI。
- **コアバリュー**: GitHub、カレンダー、ローカルファイル、そして AI エージェントを一つの画面で「観測（Intelligence）」「実行（Operations）」「報告（Logistics）」できること。

## 2. 画面構成と情報のライフサイクル

### 2.1. 観測 (Intelligence) - `HOME / BRIEFING`
一日の始まりに確認するダッシュボード画面。
- **TACTICAL STATUS**: プロジェクトの全体状況、遅延警告、システム稼働情報（API接続状態など）のモニター表示。
- **DAILY ITINERARY (新設)**: Googleカレンダーから取得した「本日のイベント（会議等）」と、「今日やるべきタスク」を統合したグラフィカルな垂直タイムライン。
- **DATALINK FEED**: 直近のアクティビティやMarkdownファイルの更新履歴のストリーム表示。

### 2.2. 実行 (Operations) - `TASKS / TACTICAL EDITOR`
タスク群の管理と、実際の執筆を行う主戦場。
- **Unified Task List**: GitHub Issues/Projects とローカルタスクを区別なく一覧化（Project/Tagでの階層化）。
- **Tactical Editor**: 2カラム・リアルタイムプレビューを備えたマークダウンエディタ。見出しのAuto-Accordion機能を搭載。
- **Metadata仕様拡張**: ローカルMarkdownのFront-matterに `status (todo/in-progress/done/icebox)`, `project`, `start_date`, `due_date`, `tags` を持たせる。

### 2.3. 計画 (Strategy) - `TIMELINE`
中長期的な視点でのスケジューリング。
- **Gantt Chart**: GitHub Projects V2 とメタデータを拡張したローカルタスクの横断ガントチャート。

### 2.4. 報告 (Logistics) - `REPORT GENERATION`
アクティビティの抽出と報告アウトプット。
- **"Reports as Documents" アプローチ**: 日報や週報は独自のUIに隔離せず、**「自動生成された特殊なMarkdownタスク」として Tactical Editor 上で展開・編集**する機能を基本とする。
- **DAILY REPORT (Operation Log)**:
  - 今日のアクティビティを自動収集し、Google Chat向けの独自Markdown方言に整形してエディタに生成。
  - エディタ上の `[ TRANSMIT ]` ボタンにより Google Chat へWebhook送信。
- **PERIODIC REPORT (Historical Record)**:
  - 任意の期間を指定し、標準Markdown形式で振り返りレポートを生成。ローカル保存主体（送信なし）。
- **EXCEL EXPORT (Special)**:
  - 指定フォーマット（作業週報Excel）への出力機能。

## 3. 外部連携と未来構想 (Integrations)

### 3.1. GitHub Integration (稼働中/改修)
- Issueからの「手動インポート」を廃止し、透過的な同期リストとしての扱いにシフトする。

### 3.2. Google Calendar Integration (新規予定)
- 予定ブロックの読み取りにより、「いつ・どのタスクをやるべきか」のスケジューリングとダッシュボード表示を行う。

### 3.3. AI Agent (MCP Server) Integration (新規構想)
- **TCC Local MCP Server**: 
  - 本アプリ自体（Electron内部）で MCP (Model Context Protocol) サーバーを立ち上げる。
  - これにより、Claude Code や Cursor, Antigravity などの**外部 AI エージェントが直接このアプリ（TCC）に接続**し、タスクの読み取り、進捗の自動更新、日報のドラフト作成などを自律的に行える「司令塔」としての機能を実現する。

## 4. アプリケーション設定 (System settings)
- **Data Location**: Markdownおよび設定ファイル群のディレクトリ指定。
- **Interface**: バージョン表記の自動化、フォントサイズ調整。
