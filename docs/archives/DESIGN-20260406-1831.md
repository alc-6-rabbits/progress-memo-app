# DESIGN-20260406-1831: HOME画面（Dashboard）の表示再設計

## 1. コンポーネント別の変更点

### 1.1 DATALINK FEED (`recentTasks`)
- **ソース/状態の表示**:
    - GitHubタスク: `#123 / repo-name (Open)` のようなヘッダーをタイトルの上に追加。
    - ローカルタスク: `Local / snippet...` をタイトルの下。
- **スタイル**:
    - フォントサイズ 10px、`opacity-60` の小文字テキストを付随。

### 1.2 Task Strategic Board
- **IceBox 列の追加**:
    - `tasksIcebox` コンポーネントを `tasksDone` の代わりに配置。
    - ステータスが `icebox` または `standby` (case-insensitive) のタスクを抽出。
- **カード内カード番号表示**:
    - `id` または `issue` (GitHub) を表示。
    - ローカルタスクの場合は、一律 `Local` のラベルを表示。

### 1.3 Tactical Timeline (`timelineTasks`)
- **フィルタ条件**:
    - `task.startDate` が存在し、かつ `task.dueDate` が存在すること。
    - いずれかが欠けているタスクは、タイムラインからは除外する。
- **バーの描画範囲**:
    - `getGanttBarStyle` 関数を修正し、`updated_at` ではなく `startDate` と `dueDate` から描画位置（`left`, `width`）を算出する。

## 2. データ構造の再確認
- `useGitHubGraphQL.js` で既に `startDate` と `dueDate` を正規化して返していることを利用。
- ローカルタスクは `matter.data` 内の `startDate` と `target_date` (リネーム検討中) または `dueDate` を使用。

## 3. UI/UX
- **色分けの明確化**:
    - GitHubアイテムとローカルアイテムで、カードのボーダー色やアイコンを微調整する。
- **動的な更新**:
    - 取得中の `pending` 状態のスキャニングアニメーションを継続利用。
