### 2.2. `.agent/workflows/commit_save.md` 改修

Issue番号のコミットメッセージ含有確認指示を追加する。

**変更前**:

```markdown
---
description: 全変更をステージングし、指定メッセージでコミットする
---
1.  **Commit Message Identification**:
    - `docs/specs/tasks/TASK-*.md` (TASK_DOC) 内の「Git運用」セクションから、指定されたコミットメッセージを確認します。
2.  **Git Commit**:
    - 変更をステージングし、指定されたメッセージでコミットします。
// turbo
3.  **Command Execution**:
    - `git add . && git commit -m "<commit_message>"`
```

**変更後**:

```markdown
---
description: 全変更をステージングし、指定メッセージでコミットする
---
1.  **Commit Message Identification**:
    - `docs/specs/tasks/TASK-*.md` (TASK_DOC) 内の「Git運用」セクションから、指定されたコミットメッセージを確認します。
    - 「関連 Issue」フィールドが存在する場合、コミットメッセージ末尾に `#N` (Issue番号) が含まれていることを確認します。含まれていない場合は自動的に追加します。
2.  **Git Commit**:
    - 変更をステージングし、指定されたメッセージでコミットします。
// turbo
3.  **Command Execution**:
    - `git add . && git commit -m "<commit_message>"`
```

### 2.3. `.agent/workflows/branch_create.md` 改修

ブランチ名にIssue番号を含める推奨を追加する。

**変更前**:

```markdown
---
description: タスク用の作業ブランチを作成しチェックアウトする
---
1.  **Branch Name Identification**:
    - `docs/specs/tasks/TASK-*.md` (TASK_DOC) 内の「Git運用」セクションから、指定されたブランチ名を確認します。
2.  **Git Checkout**:
    - 指定されたブランチ名で作業ブランチを作成し、チェックアウトします。
// turbo
3.  **Command Execution**:
    - `git checkout -b <branch_name>`
```

**変更後**:

```markdown
---
description: タスク用の作業ブランチを作成しチェックアウトする
---
1.  **Branch Name Identification**:
    - `docs/specs/tasks/TASK-*.md` (TASK_DOC) 内の「Git運用」セクションから、指定されたブランチ名を確認します。
    - ブランチ名には関連 Issue 番号が含まれていることが推奨されます（推奨形式: `{type}/{N}-{feature-name}`）。
2.  **Git Checkout**:
    - 指定されたブランチ名で作業ブランチを作成し、チェックアウトします。
// turbo
3.  **Command Execution**:
    - `git checkout -b <branch_name>`
```
