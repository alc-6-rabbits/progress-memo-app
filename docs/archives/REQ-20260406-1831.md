# REQ-20260406-1831: HOME画面（Dashboard）の表示改善

## 1. 概要
GitHub連携データの導入に伴い、実運用に合わせた表示内容の最適化を行う。情報密度の向上と、ノイズの削減を目的とする。

## 2. 具体的な要求事項

### 2.1 DATALINK FEED の拡充
- **現状**: タスクのタイトルと更新時刻のみが表示されている。
- **要求**: アクティビティの内容（ソースの判別、ステータスの変化など）が視認できるように情報を増やす。
    - GitHubタスク: リポジトリ名、Issue番号、現在のStateを表示。
    - ローカルタスク: 本文（body）のプレビュー（数文字〜一行程度）を表示。

### 2.2 Task Strategic Board の最適化
- **カードの識別子追加**:
    - 全てのタスクカードに、GitHubタスクなら `#Issue番号`、ローカルタスクなら `Local` というラベルを表示する。
- **列の入れ替え**:
    - 「COMPLETED（完了）」列はDashboardでは不要（Full Boardで見れば良いため）。
    - 代わりに、現在表示されていない 「IceBox（保留）」列を常時表示させる。

### 2.3 Tactical Timeline の表示条件変更
- **現状**: 最近のアクティビティ（実績）に基づいたタスクが表示されている。
- **要求**: 「StartDate（開始日）」と「DueDate/TargetDate（終了日）」の両方が設定されているタスクのみを表示する。
    - 実績（いつ更新されたか）ではなく、計画（いつからいつまでやるか）に基づいたタイムラインにする。

## 3. 非機能要件
- 情報量が増えても、HOME画面のフォントサイズやデザインの統一性（11px〜12px、モノスペース、TCCテーマカラー）を維持すること。
