# TASK-20260406-1831: HOME画面（Dashboard）情報の高度化

## 概要
ユーザーのフィードバックに基づき、HOME画面の各セクションの表示内容をブラッシュアップする。

## 成果物 (Deliverables)
- [ ] `pages/index.vue` の更新
- [ ] UIの動作確認（ブラウザ/ビルド）

## 実装ステップ

### 1. DATALINK FEED の情報拡充
- `recentTasks` の描画ループ内で以下の情報を追加：
    - GitHubタスク: `#No`, `Repository Name`, `State` (Open/Closedなど) を表示。
    - ローカルタスク: `Local` というラベルを表示。
    - 本文（`body`）が存在する場合、最初の1行（約30文字）をプレビューとして表示。

### 2. Task Strategic Board の再編
- **列の交換**:
    - `tasksDone` コンピューテッドプロパティを `tasksIcebox` に変更し、ステータスが `icebox` または `standby` のものを抽出。
    - テンプレート上の 「COMPLETED」タイトルを 「ICEBOX」に変更し、`tasksIcebox` をループさせる。
- **カードの識別子表示**:
    - 全カードに、タスクが GitHub 由来なら `#123` 等の番号、ローカルなら `Local` というバッジを表示。
    - `project_name` の表示を、より目立たない色（グレー）に調整。

### 3. Tactical Timeline の計画ベース化
- `timelineTasks` のフィルタを、`startDate` と `dueDate` が両方揃っているタスクのみに限定。
- `getGanttBarStyle` において、表示週（月〜日）の範囲に対する開始・終了オフセットを計算。
    - `daysBetween` 等の補助関数を用い、週を跨ぐタスクのクリッピング処理を実装（v1.4.1 Porting）。
    - バーの色は `scheduleStatus` に基づく（on-track: 青, at-risk: 黄, overdue: 赤）。
