# Progress Memo App
**Version 1.5.0 "Weekly Report & Architecture Revamp"**

Progress Memo App は、Electron と Nuxt 3 を基盤として構築された、ローカル完結型のパーソナルタスク・進捗管理デスクトップアプリケーションです。
サイバーパンクやフライトシューティングゲーム（『エースコンバット7』など）の戦術データリンク（UI/UX）にインスパイアされた、ダークブルー基調の情報密度の高いデザインを採用しています。

## 🖥 動作環境・技術スタック
- **Host / Window Manager**: Electron (v34) - `fs` 経由でのローカルファイルダイレクトアクセス
- **Frontend Framework**: Nuxt 3 (Vue 3 / Vite)
- **Styling**: Tailwind CSS (独自拡張のAce Theme適用)
- **Data Format**: Markdown (Front-matter 適用によるメタデータ管理) + JSON

## ✨ 主な機能

### 1. 「DATALINK」ベースのタスク・ドキュメント管理 (TASKS画面)
すべてのタスクはローカルの `.md` ファイルとして保存され、Nuxt側でMarkdownコンテンツとしてリアルタイムパースされます。
- **タブ管理**: タスクの鮮度に合わせて「In Progress（進行中）」「Icebox（保留）」「Archived（完了/保管）」の3つのステータスで表示を切り替え。
- **自動ミュート機能**: Icebox の中で30日以上更新がない非アクティブなタスクは標準で非表示にし、「SHOW INACTIVE」トグルで一括表示可能。
- **タググルーピング**: 新機能「BY TAG」トグルにより、プロジェクト単位の表示を、タグによる2段階階層（Tag > Project > Task）に切り替えて表示可能。
- **外部エディタ連携**: `[ RELOAD DATALINK ]` ボタンにより、VSCode等で直接マークダウンファイルを編集した際も瞬時にアプリ内に同期。

### 2. 進化した 2カラム フルスクリーン Editor
- **画面分割・リアルタイムプレビュー**: 編集ボタン（`>> EDIT CONTENT`）から、左側にエディタ、右側にHTMLプレビュー画面を展開。
- **Markdown ツールバー機能**: GitHubライクに、H1〜H3、太字、斜体、引用、コードブロック、リンク、リスト（箇条書き・番号付き・チェックボックス）などをワンボタンで挿入可能。
- **自動アコーディオン (Auto-Accordion)**: プレビュー側にある見出し（Heading 1〜6）要素をクリックすると、次の同レベルの見出しまでを動的に折りたたむ（開閉）ことが可能。HTMLタグを書かずともJavaScript制御で実現。

### 3. プロジェクトマネージャー機能 (PROJECTS画面)
- 個別のタスク（事案）を束ねる「プロジェクト」の概念を導入。`projects.json` によって一元管理。
- TASKS画面では、指定したプロジェクトごとにタスクがグルーピングされて表示されます（未指定の場合は `UNCLASSIFIED` として分類）。
- **タグ機能**: 各プロジェクトに任意のタグ（複数可）を付与でき、TASKS画面でのフィルタリングやグルーピングに活用可能。
- TASKS編集パネルや新規作成フローにて、追加済みのプロジェクトリストをプルダウンアシストとして利用可能。

### 4. 日報・レポート連携 (REPORT画面)
- **マークダウンエディタ**: エディタ部を刷新し、フリーテキストのマークダウン形式で日報を記述可能。
- **ツールバー & テンプレート**: 太字やリストなどの装飾ボタンに加え、日報・週報の定型文テンプレートをワンクリックで挿入可能。
- **週報作成・保存機能**: 指定した期間（月〜金）のタスク更新および GitHub アクティビティを自動抽出し、雛形として挿入。完成した週報はローカルディレクトリ (`weekly-reports/`) へファイルとして保存。
- **コンテキストメニュー**: 右クリックから現在アクティブなプロジェクト名やタスク名を即座に挿入。
- **Google Chat 同期**: 登録した Webhook URL を通じてダイレクトにチャットルームへ送信。
- **GitHub Issue 自動コメント**: レポート送信時、紐付けられた GitHub Issue へ進捗を知らせる自動コメントを投稿可能。
- **アキュムレーションの廃止**: 従来の「データ蓄積」ベースのフローを廃止し、タスクからの直接集計方式に完全移行。

### 5. 統合ダッシュボード & ガントチャート (BRIEFING & TIMELINE画面)
- **MISSION BRIEFING (ダッシュボード)**: ローカルのタスクと GitHub Projects V2 の状況を横断的に把握できる統合画面。カンバンボード形式のステータス概要やアラート通知機能を提供。
- **OPERATION TIMELINE (ガントチャート)**: GitHub Projects V2 の開始日・期限日をベースに、横スクロール可能なタイムライン上でアイテムのスケジュールを可視化。遅延タスクの赤色ハイライト等に対応。
- **GitHub Activity 自動取得**: REPORT 画面のコンテキストメニューから、当日の GitHub アクティビティ（コミット、Issue の作成・コメントなど）を日報へ自動挿入可能。

### 6. 保存先ディレクトリの自由設定 (SETTINGS画面)
- Electron のネイティブ機能（フォルダピッカー）を活用し、タスク用MarkdownファイルおよびプロジェクトJSONを保存する **ルートディレクトリを自由に設定可能**。
- `G:\My Drive\...` のような Google ドライブマウント領域や OneDrive などを指定することで、クラウド越しのバックアップ環境をシームレスに構築できます。

### 7. フォントサイズ調整機能 (SETTINGS画面 & ショートカット)
- アプリ全体のフォントサイズ（12px 〜 32px）を動的に変更可能。
- **ショートカットキー**:
  - `Ctrl` + `+` / `=` : 拡大 (+2px)
  - `Ctrl` + `-` : 縮小 (-2px)
  - `Ctrl` + `0` : リセット (16px)
- SETTINGS画面の「INTERFACE & STARTUP」セクションからも数値を直接指定して変更・保存が可能です。

### 8. 外部連携設定 (SETTINGS画面)
- **GitHub PAT (Personal Access Token)**: Issue への自動コメント機能、Projects V2 の同期、Issue のインポートに使用。
- **Google Chat Webhook**: 日報の直接送信用。
- **GitHub Issue 自動コメントトグル**: 日報送信時の自動投稿をグローバルに ON/OFF 可能。
- **起動画面の指定**: アプリ起動時の初期画面を TASKS / REPORT / BRIEFING / TIMELINE から選択可能。

## 🚀 開発（起動）方法
このプロジェクトは開発段階です。ソースコード取得後、以下のコマンドで起動します。

```bash
cd progress-memo-app
npm install
npm run dev
```

> **Note**: アプリケーションメニューは非表示に設定されています。開発者ツール (DevTools) を開きたい場合は、アプリ内で `F12` キーを押してください。
