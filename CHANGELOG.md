# CHANGELOG

## [1.7.3] - 2026-04-08
### Fixed
- **データ保存先ディレクトリ設定の完全同期**
  - Settings 画面で指定した「Database Root Path」が、タスクおよびレポートの一覧取得時にも正しく反映されるように修正。
  - フロントエンド（Home, Tasks, Reports 画面）からデータ取得 API 呼び出し時に、設定済みのパスを確実に渡すように改善。
- **レポート一覧の読み込み不具合の修正（Windows環境対応）**
  - メインプロセスでのファイル一覧取得において、Windows のパス区切り文字（バックスラッシュ）と干渉しやすかった `fast-glob` を廃止し、標準の `fs.promises.readdir` に変更。
  - これにより、カスタムパス設定時にレポートが表示されない問題を解消。

## [1.7.2] - 2026-04-07
### Fixed
- **GitHub アクティビティ取得不具合の修正（タイムゾーン対応）**
  - 日本時間の深夜帯（22:00以降など）に「今日のアクティビティ」が取得できなくなる問題を修正。
  - タイムゾーンの影響を受けにくい日付生成ロジック (`parseLocalDate`) を導入し、JST/UTC の解釈不整合を解消。
  - 取得開始時刻に1時間のバッファを設け、システムクロックの微小なずれによる漏れを防止。
  - 最大取得件数を 500件から 1000件（10ページ分）へ拡張。
- **GitHub 取得エラーの可視化改善**
  - PAT の設定ミス（Authorize 漏れ含む）や API 制限などで取得に失敗した際、レポート画面内に詳細なエラーメッセージを表示するように改善。
  - 診断用の内部ログ出力を強化し、トラブルシューティングを容易化。

## [1.7.1] - 2026-04-07
### Fixed
- **報告書（日報・週報）の保存先不具合の修正**
  - Settings で指定したカスタムディレクトリが正しく反映されず、意図しない場所に保存される問題を修正。
  - レンダラープロセスからメインプロセスへ正しく保存先パスを渡すように改善。
- **報告書フォーマットの微調整**
  - GitHub Project 連携時の箇条書き記号を `・` からマークダウン標準の `- ` に変更。
  - プロジェクト名と進捗状況セクションの間に空行を挿入し、視認性を向上。
  - 各タスク行の末尾に半角スペース2つ（マークダウン改行）を追加。
  - 追記用記号 `⇒` の前後に半角スペース2つを配置し、段落の整列を改善。

## [1.7.0] - 2026-04-06
### Added
- **HOME画面（Dashboard）の表示高度化 (Evolution Phase)**
  - DATALINK FEED: GitHub（#No/Repo）およびローカル（本文プレビュー）の詳細表示に対応。
  - Task Board: 「COMPLETED」列を「ICEBOX」へ変更し、保留タスクの可視性を向上。
  - Task Board: 各カードに識別子バッジ（#No / Local）を追加。
  - Tactical Timeline: 計画ベースの実装へ刷新。`startDate` と `dueDate` (Target Date) に基づく期間表示に対応。

### Fixed
- **HOME画面のGitHub連携データの表示機能の復元**
  - 最近のバージョンで消失していた GitHub Projects V2 からのデータ取得・統合ロジックを再実装。
  - `useGitHubGraphQL` を Dashboard (`index.vue`) に統合し、ローカルタスクと GitHub アイテムの並列取得・最新順ソートを実現。

## [1.6.0] - 2026-04-05
### Added
- **プロジェクト進捗状況の可視化 (GitHub Projects 連携)**
  - サブ課題（Sub-issues）の完了状況を集計し、完了数/合計/進捗率(%)を自動表示する機能を追加。
  - Sub-issues 機能が利用不可、または権限がない場合のフォールバック表示（Sub-Issueなし）を実装。
- **報告書生成 UI & フォーマットの改善**
  - 週報生成時の「期間指定モーダル」を追加（flatpickr を使用）。デフォルトで当週の月〜金を自動セット。
  - 各タスク行の直下に、内容を追記しやすくするための記号（`⇒`）を自動挿入。
  - 「明日/来週の予定」セクションにおける優先度（Priority）および期限（Due Date）ベースのソートロジックを導入。
  - 日報・週報の生成テンプレート処理を共通化。
- **GitHub 非連携時の動作保証 (Graceful Degradation)**
  - GitHub PAT 未設定、または API 呼び出し失敗時でもローカルタスクのみで報告書を生成できるようガード。

## [1.5.0] - 2026-03-25
### Added
- **週報作成・保存機能の追加**
  - REPORT 画面にて、指定期間（月〜金）の活動をまとめた週報を自動生成し、ローカルファイル (`weekly-reports/`) として保存する機能を追加。
- **GitHub アクティビティのソート順改善**
  - レポートに挿入される GitHub アクティビティを Issue ナンバーの昇順でソートするように変更。

### Removed
- **アキュムレーション機能の廃止**
  - 旧来の「データ蓄積（Accumulation）」ベースのワークフローを廃止。タスクファイルからの直接集計方式へ移行。
  - 関連する IPC ハンドラーおよび UI 要素（ACCUMULATED LOGS タブ等）を完全に削除。
